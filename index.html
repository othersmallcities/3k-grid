The Grid

Both, large and small grid come with 12 columns, but break at different sizes. Please change the window size to see the effect.

All variables can be found in "import/_layoutBase.scss".

Please note that the class col has to be added to each column, otherwise it will not work.

Just checkout the code examples on the right.

Centered Columns

Centred columns are only that, columns which should be used to centre content. You can add the class of .tCent to centre the content of the column as well.
Please note that only one centred column per row should exist, but that is up to you.

<article class="row">
  <p class="s6 centCol">.s6 .centCol</p>
</article>
<article class="row">
  <p class="l7 centCol">.l7 .centCol</p>
</article>

Source ordered Columns

Source ordered columns can be very useful if you want to display secondary information before primary information, for example: Display a navigational sidebar on the left and the main content on the right.
For SEO and screen readers it is better to write the main content first and let the code for the sidebar follow.
You must activate source ordering in the each layout file, starting with _smallGrid.scss.
You have to declare both the normal row width class and the push / pull class.
Please improve this, if you have a good idea.

<article class="row">
  <p class="ph-8 l4 col">.ph-8 .l4 .col</p>
</article>
<article class="row">
  <p class="ph-4 l8 col">.ph-4 .l8 .col; last</p>
</article>

ph-8, l4

pl-4, l8; last

ph-2, l10

pl-10, l2; last

sPh-8, s4

sPl-4, s8, last

sPh-2, s10

sPl-10, s2; last

You can activate source ordering for small grids only, large grids require the small grid file, since all push (.ph & .sph) and pull (.pl & .spl) classes are stored there.
Change it as needed.

I have seperated them into large and small to enable you to push and pull differently on each size if needed. If you see a need to save some bytes and do not need one or the other class, just disable what you do not need.

A simple block grid

The block grid is specially made for displaying blocks of images.
I am not yet 100% happy with how it works. Check the code if you needed and improve on it, please.
Like the source ordering it has to be activated separately, staring in _layoutBase.scss. The large block grid also depends on the small block grid, just like source ordering.
The idea comes from here, though I have greatly simplified it.
You can add 6 items (.bGrid2), 4 items (.bGrid4) or 2 items (.bGrid6) per row with auto reflow.
You can activate the small version independently of the large version.
